一个面向实际生产环境、大数据量站点的 WordPress 备份与还原工具。  
**核心目标：速度优先、稳定优先、可控优先。**

> ⚠️ **注意：** 本项目为高性能优化版，直接调用系统级命令，不追求在受限虚拟主机上的极端兼容性。

---

## ✨ 核心特性

### ⚡ 极致性能
- **系统级执行**：优先使用 `exec` / `popen` 调用系统指令，避开 PHP 内存限制。
- **数据库直导**：直接调用 `mysql` 客户端导入 SQL，效率远高于 PHP 逐条执行。
- **文件原地覆盖**：使用系统级解压与移动，避免低效的 PHP `copy` 循环。

### 🔄 智能还原引擎
- **异步后台处理**：任务在后台 Worker 进程执行，绕过 Cloudflare 100秒响应限制。
- **分步还原策略**：自动拆分解压、数据库导入、域名替换等阶段，规避 PHP 超时。
- **自动域名替换**：还原时自动将备份中的旧域名替换为当前域名（深度支持递归序列化数据）。

### 🛠️ 生产级可靠性
- **Session 保持**：还原后自动恢复当前用户的登录状态（同域名下 100% 不掉线）。
- **实时进度反馈**：前端模态窗显示精确进度，并保留详细的调试日志。
- **安全存储**：备份存放在 `wpbkres` 目录，通过 `.htaccess` 自动禁止 Web 直接访问。

## ⚙️ 运行环境要求

### ✅ 基础要求
- **PHP ≥ 7.4**
- **MySQL / MariaDB** 客户端工具可用
- 站点目录具有 **写权限**

### ✅ 必须支持的函数
为了发挥高性能，以下函数在 `php.ini` 中**不能**被禁用：
- `exec`, `popen`, `proc_open`
- `curl` 或 `wget`

### ❌ 不支持的环境
- 禁用 `exec` 的受限虚拟主机（Shared Hosting）
- 无法执行系统命令的极度加固环境
- *本项目不提供纯 PHP 降级方案，以确保大站点恢复的成功率。*

## 📦 备份文件结构

生成的 `.bgbk` 文件实质为一个标准的压缩包，结构如下：
```text
backup.bgbk
├── database.sql    # 数据库导出文件
├── files.zip       # 站点文件打包
└── meta.json       # 元信息（版本、域名、校验码、时间戳）
```

## 🚀 使用说明

### 🔹 备份流程
1. 进入「工具」→「WP备份还原」。
2. 点击 **「开始高性能全站备份」**。
3. 插件自动扫描并打包，生成 `.bgbk` 单文件。
4. 备份完成后可直接在列表下载或管理。

### 🔹 还原流程
1. 从下拉列表中选择备份文件，点击 **「开始还原」**。
2. 插件将自动执行：**环境检测 -> 解压 -> 文件恢复 -> 数据库导入 -> 域名替换 -> 状态恢复**。
3. 还原完成后，点击「确定并手动刷新」即可。

### 🔹 调试与日志
- 日志级别可调（OFF/ERROR/WARNING/INFO/DEBUG）。
- **实时监控：**
  ```bash
  tail -f wp-content/uploads/wpbkres/restore.log
  ```

## 📊 还原进度阶段说明

| 阶段 | 进度权重 | 关键操作 |
| :--- | :--- | :--- |
| **初始化** | 0% - 10% | 环境检测、权限检查、临时目录准备 |
| **解压** | 10% - 45% | 解压 `files.zip` 并原地覆盖现有文件 |
| **数据库** | 45% - 85% | 调用 `mysql` 导入 `database.sql`，执行事务提交 |
| **配置清理** | 85% - 100% | 自动替换 `siteurl`/`home`，重建 Session，清理临时文件 |

## ⚠️ 注意事项

1. **覆盖风险**：还原操作会彻底覆盖当前站点的文件与数据库，操作前请确认。
2. **磁盘空间**：请确保服务器剩余空间至少为站点体积的 1.5 倍。
3. **数据库优化**：如遇 SQL 报错，请尝试增大 MySQL 的 `max_allowed_packet`（建议 256M）。
4. **固定链接**：还原后若出现 404，请前往「设置 → 固定链接」点击一次保存。

## 🛠️ 故障排查

- **进度条卡在 0%？** 检查浏览器控制台 AJAX 请求是否被防火墙拦截，或检查 `exec` 是否被 PHP 禁用。
- **还原提示“环境不达标”？** 请确认 `popen` 函数可用，且系统临时目录可写。
- **还原后需要重新登录？** 仅当跨域名还原或浏览器 Cookie 策略限制时发生。
