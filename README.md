一个面向实际生产环境、大数据量站点的 WordPress 备份与还原工具。  
**核心目标：速度优先、稳定优先、可控优先。**

> ⚠️ **注意：** 本项目为性能优化版，直接调用系统级命令，不追求在受限虚拟主机上的极端兼容性。

---

## ✨ 核心特性

### 🚀 极致性能
- **系统级执行**：优先使用 `exec` / `popen` 调用系统指令，避开 PHP 内存限制。
- **数据库直导**：直接调用 `mysql` 客户端导入 SQL，效率远高于 PHP 逐条执行。
- **文件原地覆盖**：使用系统级解压与移动，避免低效的 PHP `copy` 循环。

### 📦 智能分片上传 (NEW!)
- **动态分片**：根据网络状况自动调整分片大小（512KB - 10MB）。
- **断点续传**：支持记录已上传区间，刷新页面后可继续上传。
- **指数退避重试**：上传失败时自动延迟重试，确保极高成功率。

### 🔄 智能还原引擎
- **异步后台处理**：任务在后台 Worker 执行，绕过 Cloudflare 100秒响应限制。
- **分段还原策略**：自动拆分解压、数据库导入、域名替换等阶段，规避 PHP 超时。
- **自动域名替换**：还原时自动将旧域名替换为新域名（深度支持递归序列化数据）。

### 🛠️ 生产级可靠性
- **Session 保持**：还原后自动恢复当前用户的登录状态（同域名下 100% 不掉线）。
- **实时进度反馈**：现代化的 UI 界面，提供精确到 SQL 行数的进度追踪。
- **安全存储**：备份存放在 `wpbkres` 目录，通过 `.htaccess` 自动禁止 Web 直接访问。

## ⚙️ 运行环境要求

### ✅ 核心服务器配置 (重要)
为了确保大数据量备份与分片上传的稳定性，请务必在服务器上配置以下参数：

**PHP 配置 (`php.ini`):**
```ini
upload_max_filesize = 512M
post_max_size = 512M
memory_limit = 512M
max_execution_time = 0
```

**Nginx 配置 (`nginx.conf`):**
```nginx
client_max_body_size 512M;
```

### ✅ 基础要求
- **PHP ≥ 7.4**
- **MySQL / MariaDB** 客户端工具可用。
- 必须支持函数：`exec`, `popen`, `proc_open`, `curl` 或 `wget`。

## 🚀 使用说明

### 🔹 备份流程
1. 进入「工具」→「WP备份还原」。
2. 点击 **「立即备份」**。
3. 插件自动扫描并打包，生成 `.bgbk` 单文件。

### 🔹 上传备份
1. 点击 **「上传备份」** 选择本地的 `.bgbk` 文件。
2. 插件将启动**动态分片上传**，进度条实时显示上传百分比。
3. 若上传中断，重新选择同一文件即可实现**断点续传**。

### 🔹 还原流程
1. 从下拉列表中选择备份文件，点击 **「全站还原」**。
2. 插件将自动执行：环境检测 -> 解压 -> 文件恢复 -> 数据库导入 -> 域名替换 -> 状态恢复。
3. 还原完成后，点击「确定并刷新页面」即可。

## 🛠️ 故障排查

- **进度条卡在 0%？** 检查浏览器控制台 AJAX 请求是否被防火墙拦截，或检查 `exec` 是否被 PHP 禁用。
- **上传速度慢或失败？** 插件会自动降级分片大小进行重试。请确保 Nginx 的 `client_max_body_size` 已设为 512M。
- **还原后 404？** 请前往「设置 → 固定链接」点击一次保存，重新生成 `.htaccess`。
- **实时监控日志：**
```bash
tail -f wp-content/uploads/wpbkres/restore.log
```
