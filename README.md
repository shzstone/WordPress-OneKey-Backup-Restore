# WP 一键备份还原

**WP 一键备份还原** 是一款高性能的 WordPress 备份与还原插件，支持异步备份、主动分步还原、自动域名替换，并提供实时进度反馈。特别适合大数据量站点（数万文件、数百万数据）的迁移与恢复，确保数据完整性与操作流畅性。

## ✨ 主要特性

- **一键备份**：自动打包网站所有文件与数据库，生成独立的 ZIP 备份文件。
- **一键还原**：支持从备份文件快速还原，自动覆盖现有文件与数据。
- **异步处理**：备份任务在后台运行，不阻塞用户操作。
- **分步还原**：自定义格式备份支持分步还原，避免 PHP 执行超时。
- **自动域名替换**：还原时自动将备份中的旧域名替换为当前站点域名（包括数据库中的 `siteurl`、`home`、文章内容等）。
- **实时进度**：前端显示备份与还原进度（阶段式更新），同时保留详细日志便于排查。
- **兼容性**：支持 ZIP 格式备份与自定义格式（`.bgbk`），兼容主流服务器环境。
- **安全存储**：备份文件存放在 `wp-content/uploads/wpbkres/` 目录，通过 `.htaccess` 阻止直接访问。

## 📦 安装

1. 下载插件源码，将整个文件夹上传至 WordPress 的 `/wp-content/plugins/` 目录。
2. 在 WordPress 后台「插件」页面找到“WP 一键备份还原”，点击「启用」。
3. 进入「工具」→「WP备份还原」即可使用。

---

## 🚀 使用方法

### 备份
- 设置每批文件数和数据库行数（默认值已优化，一般无需修改）。
- 点击「开始备份」，插件会自动扫描文件并打包，过程中会显示进度。
- 备份完成后，备份文件会出现在下拉列表中。

### 还原
1. 从下拉列表中选择需要还原的备份文件。
2. 设置每批还原条目数（建议 500~2000，视服务器性能而定）。
3. 点击「开始还原」，插件会依次解压文件、导入数据库、替换域名。
4. 还原期间可通过前端模态框查看阶段进度（解压中 → 导入中 → 替换域名 → 完成）。

## 📊 进度与日志

- **前端进度**：还原过程中会显示阶段百分比（如 20% → 50% → 95% → 100%），但部分服务器环境下可能无法实时刷新，建议同时关注后台日志。
- **日志位置**：插件会在 `wp-content/uploads/wpbkres/restore.log` 中记录详细操作信息，可通过 `tail -f` 命令实时查看。
- **验证还原**：还原完成后，可登录网站后台检查内容，或查看日志中的“SQL 导入完成，共执行 x 条语句”确认数据完整性。

## ⚠️ 注意事项

1. **备份文件安全**：备份目录默认通过 `.htaccess` 禁止 Web 访问，请勿随意修改权限。
2. **执行时间与内存**：对于超大站点，建议在 PHP 配置中提高 `max_execution_time` 和 `memory_limit`。
3. **自定义格式备份**：`.bgbk` 格式是早期版本遗留，新备份默认使用 ZIP 格式，更通用且还原更快。
4. **域名替换**：还原时会自动将备份中的 `siteurl` 和 `home` 替换为当前域名，并更新文章与元数据中的链接。
5. **菜单丢失**：如还原后导航菜单消失，请进入「外观 → 菜单 → 管理位置」重新分配菜单。

## 🛠️ 常见问题

### Q：还原时进度条卡在 0% 怎么办？
A：请检查浏览器控制台是否有 AJAX 错误，或查看 PHP 错误日志。如果日志显示正在解压或导入，说明任务正常执行，前端可能因网络或缓存未及时更新，请以日志为准。

### Q：导入时出现 “MySQL server has gone away” 错误？
A：通常是由于 SQL 文件过大或超时时间过短。插件已设置会话超时和包大小，若仍出现，可尝试调低“还原每批条目数”，或在 MySQL 配置中增大 `max_allowed_packet` 和 `wait_timeout`。

### Q：备份时提示“无法创建临时目录”？
A：请检查 `wp-content/uploads/` 目录是否可写，或手动创建 `wpbkres` 目录并设置 755 权限。
