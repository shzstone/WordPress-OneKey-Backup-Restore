这是一个专为**大数据量站点**设计的 WordPress 备份与还原工具。不同于常规插件，它直接调用系统级能力，追求极致的执行速度与还原成功率。

> ⚠️ **运行前提**：服务器需支持 `exec` / `popen` 函数（VPS/云服务器通常均支持）。

---

## ✨ 核心特性 (v1.0.3)

### 🛡️ 智能预检与安全 (New)
- **磁盘空间校验**：在备份/还原前自动计算站点体积，确保磁盘空间充足（预留 20% 冗余），避免中途因空间耗尽导致崩溃。
- **ZIP64 风险预警**：针对 4GB 以上的超大备份文件，自动检测 PHP 与 `libzip` 环境的兼容性。
- **UI 交互优化**：增加了空间计算时的 Loading 等待状态，大站点操作不再“假死”。

### 🚀 极致传输性能
- **动态分片上传**：支持 512KB - 10MB 自动调节的分片上传逻辑。
- **断点续传**：支持网络中断后从已上传位置继续，无需重头开始。
- **指数退避重试**：内置重试机制，应对弱网环境表现极佳。

### 🔄 高可靠还原引擎
- **原子级数据库切换**：采用“临时表导入 + 原子重命名”技术。还原过程中若失败，原站点数据依然完整。
- **深度序列化替换**：完美处理 WordPress 复杂的序列化域名替换，确保插件配置与菜单不损坏。
- **Session 保持**：还原后自动恢复当前管理员登录状态，100% 不掉线。

## ⚙️ 服务器环境配置

为了发挥本插件的最高性能，请确保以下参数已配置（建议值 512M）：

**PHP (`php.ini`):**
```ini
upload_max_filesize = 512M
post_max_size = 512M
memory_limit = 512M
max_execution_time = 0
```

**Nginx:**
```nginx
client_max_body_size 512M;
```

## 🚀 使用指南

### 1️⃣ 备份
- 进入「工具」→「WP备份还原」。
- 点击 **「立即备份」**。系统会先进行磁盘空间预检，确认无误后开始。
- 生成的 `.bgbk` 文件包含全站文件、数据库 SQL 以及站点元数据。

### 2️⃣ 上传
- 若需迁移网站，可直接点击 **「上传备份」**。
- 支持超大文件，若上传中断，重新选择同一文件即可触发**断点续传**。

### 3️⃣ 还原
- 从列表选择备份文件，点击 **「全站还原」**。
- 系统会进行空间与 ZIP64 兼容性预检。
- 还原流程：环境检测 -> 解压文件 -> 临时表导入 -> 原子切换 -> 域名替换 -> 状态恢复。

## 📊 还原进度说明

| 阶段 | 进度权重 | 关键操作 |
| :--- | :--- | :--- |
| **预检阶段** | 0% - 10% | 空间计算、ZIP64 检测、权限核验 |
| **解压阶段** | 10% - 45% | 系统级解压，原地覆盖现有文件 |
| **数据库导入** | 45% - 85% | 临时表批量导入（不影响原站点运行） |
| **原子切换** | 85% - 100% | 重命名数据表、域名递归替换、Session 重建 |

## 📁 目录与安全
- **数据存储**：`wp-content/uploads/wpbkres/`
- **安全加固**：内置 `.htaccess` 禁止 Web 直接访问备份包。
- **实时监控**：可通以下命令查看详细执行日志：
```bash
tail -f wp-content/uploads/wpbkres/restore.log
```

---

## 🛠️ 故障排查
1. **进度条卡在 0%**：通常是 AJAX 请求被防火墙拦截，或 `exec` 函数被禁用。
2. **空间报错**：还原需要站点体积约 2-2.5 倍的剩余空间（含备份压缩包）。
3. **404 错误**：还原后请前往「设置 → 固定链接」点击一次保存以刷新 `.htaccess`。
